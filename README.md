# 家事代行サービス LP

## はじめに

本リポジトリは、家事代行サービスのランディングページ（LP）です。

※本サイトは実際に運営されている猫専門ペットシッターサービスを参考に制作しています。掲載内容および制作については、サービス提供者ご本人の許可を得ています。

## 制作背景

参考にした猫専門ペットシッターサービスでは、ペットシッターだけでなく家事代行サービスも提供しています。

実際にサービス提供者へヒアリングを行った際、

「猫シッターだけでなく家事代行サービスも提供することで集客率の向上を図りたい」
「サービスの幅を広げることで、お客様とのご縁をより大切にしていきたい」

というお話を伺いました。

しかし、家事代行サービス専用のWebサイトは存在しておらず、サービス内容を十分に伝えられる環境が整っていませんでした。

そこで、既存の猫シッターサイトからもアクセスできる家事代行サービス専用のランディングページを制作し、サービスの認知向上と利用促進につなげることを目的としました。

また、初めて利用する方でも安心してサービス内容や料金を確認できるよう、分かりやすい情報設計と見やすいデザインを意識して制作しました。

## コンセプト

「高齢者の方でも見やすく、迷わず利用できるサイト」

実際にサービス提供者へヒアリングを行い、利用者層やサービスの特徴を把握した上でデザインを設計しました。

高齢者の方でも直感的に操作できるよう、文字サイズや余白を大きめに設定し、視認性を意識しました。

また、ボタンやリンクはホバー前とホバー後で色のコントラストを大きく変え、状態の変化が分かりやすいデザインを採用しました。

* ホバー前：背景色を薄く、文字色を濃く設定
* ホバー時：背景色を濃く、文字色を薄く設定

利用者がクリック可能な要素を直感的に認識できるよう工夫しています。

## 苦労した点

* レスポンシブ対応

レスポンシブデザインの実装は初めての経験だったため、参考書やChatGPTを活用しながら学習を進めました。

コードをそのまま利用するのではなく、「なぜそのコードが必要なのか」「どのような仕組みで動作しているのか」を理解することを意識しました。

* デバッグ

不具合が発生した際は、以下の手順で原因を特定しました。

1. デベロッパーツールでエラーを確認する
2. HTML・CSS・JavaScriptの読み込み状況を確認する
3. console.logを利用して処理の実行状況を確認する
4. イベントが正常に発火しているか確認する
5. HTML要素が正しく取得できているか確認する
6. 要素やコードを一つずつ削除しながら原因を切り分ける
7. 修正後に再度動作確認を行う
